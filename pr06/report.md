# ПР №6. Анализ сетевого трафик: tcpdump и Wireshark

## 1. Разбор строки tcpdump
| Поле              | Значение                  | Что означает |
|-------------------|---------------------------|--------------|
| 12:34:56.789012   | 12:34:56.789012           | Время получения пакета (с точностью до микросекунд) |
| IP                | IP                        | Протокол сетевого уровня (IPv4) |
| 127.0.0.1.44321   | 127.0.0.1.44321           | IP-адрес и порт источника (localhost) |
| >                 | >                         | Направление передачи |
| 127.0.0.1.8080    | 127.0.0.1.8080            | IP-адрес и порт назначения |
| Flags [S]         | Flags [S]                 | TCP-флаги: SYN (установка соединения) |
| seq 123456789     | seq 123456789             | Начальный номер последовательности |
| length 0          | length 0                  | Длина полезной нагрузки в байтах |

## 2. Перехваченные данные HTTP

![Пароль в открытом виде](screens/http-password.png)

**Что видно в трафике:**
- Логин: `admin` / `vasya`
- Пароль: `supersecret123` / `MyPassword2024`
- Полное тело POST-запроса в открытом виде

**Кто в реальной сети может так же прочитать этот трафик:**
- Любой, кто находится в одной сети (при использовании хаба или ARP-spoofing)
- Администратор сети
- Злоумышленник, выполнивший MitM-атаку
- Владелец Wi-Fi (если сеть не защищена)

## 3. Follow TCP Stream

![Follow TCP Stream](screens/follow-stream.png)

**Что видит злоумышленник, перехвативший этот трафик:**
Злоумышленник видит **полный диалог** между клиентом и сервером в читаемом виде: все HTTP-заголовки, тело запроса с логином и паролем, ответы сервера. Все чувствительные данные полностью скомпрометированы.

## 4. HTTP vs HTTPS — сравнение

| Что наблюдаем             | HTTP (порт 8080)               | HTTPS (порт 443)                     |
|---------------------------|--------------------------------|--------------------------------------|
| IP-адреса сторон          | Видны                          | Видны                                |
| Порт назначения           | Виден (8080)                   | Виден (443)                          |
| Заголовки HTTP            | Видны                          | Не видны (зашифрованы)               |
| Содержимое запроса (body) | Видно открытым текстом         | Зашифровано                          |
| Логин и пароль            | Видны                          | Не видны                             |
| Имя домена (SNI)          | Видно в Host:                  | Видно в Client Hello (SNI)           |

## 5. DNS и приватность

**Какие домены видны в DNS даже при HTTPS:**
- `example.com`
- `github.com`
- `google.com`

**Что это означает для приватности:**
Даже при использовании HTTPS провайдер, администратор сети или злоумышленник может видеть, **какие сайты** посещает пользователь. Это серьёзная утечка метаданных. Для повышения приватности рекомендуется использовать **DNS over HTTPS (DoH)** или **DNS over TLS (DoT)**.
