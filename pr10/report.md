# ПР №10. Лицензирование и open-source compliance

## 1. Выбор лицензии

Выбранная лицензия: MIT

Тип: разрешительная

Что разрешает:

использовать код;
копировать и распространять;
изменять и создавать производные работы;
включать в коммерческие продукты;
продавать программное обеспечение.

Что требует от пользователей кода:

сохранить уведомление об авторских правах;
сохранить текст лицензии MIT.

Может ли кто-то продавать ваш код:
Да. Лицензия MIT позволяет использовать код в коммерческих целях, включая его продажу, при условии сохранения уведомления об авторстве и текста лицензии.

## 2. Результат licensee detect

Вывод команды:

```text
License: MIT
Match: XX%
Confidence: High
```

Что означает Match %:
Match показывает степень совпадения файла LICENSE с эталонным текстом лицензии из базы данных Licensee. Чем ближе значение к 100%, тем точнее определена лицензия.

Откуда GitHub знает какая у репозитория лицензия:
GitHub использует механизм Licensee, который анализирует файл LICENSE и сравнивает его с базой известных лицензий.

## 3. SPDX-заголовки

Пример заголовка который добавили в файл:

```text
SPDX-FileCopyrightText: 2026 Никитенко Павел <yasmotruiuzaki@gmail.com>
SPDX-License-Identifier: MIT
```

Что означает каждая строка:

SPDX-FileCopyrightText: указывает владельца авторских прав и год создания файла.
SPDX-License-Identifier: указывает лицензию файла в стандартизированном формате SPDX.

Результат reuse lint:

```text
Congratulations! Your project is compliant with version X.X of the REUSE Specification.
```

Сообщение означает, что проект соответствует требованиям спецификации REUSE.

## 6. Анализ совместимости

Сценарий 1 (MIT + GPL в коммерческом):
Использовать GPL-библиотеку в закрытом коммерческом продукте нельзя без выполнения требований GPL. Придётся открыть исходный код продукта.

Сценарий 2 (Apache-2.0 + MIT):
Совместимо. MIT можно использовать в проекте под Apache-2.0.

Сценарий 3 (MIT без указания авторства):
Нарушение лицензии MIT. Требуется сохранить уведомление об авторских правах и текст лицензии.

Сценарий 4 (AGPL в SaaS):
AGPL требует предоставления исходного кода пользователям сервиса даже при использовании программы через сеть. Закрытый SaaS без раскрытия исходников невозможен.

Сценарий 5 (нет LICENSE):
Использовать такой код нельзя. По умолчанию действуют все авторские права автора.
