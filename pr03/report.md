# ПР №3. Права доступа Linux и управление пользователями

## 1. Пользователи и группы

Созданные пользователи и их роли:
| Пользователь | Группа | Роль |
|---|---|---|
| alice | developers | Администратор проекта |
| bob | developers | Разработчик |
| carol | auditors | Аудитор |

Поля /etc/passwd (разбор строки alice):
Строка имеет 7 полей:
Имя пользователя — alice
Пароль x (хранится в /etc/shadow)
UID идентификатор пользователя
GID идентификатор основной группы
Комментарий (GECOS) описание пользователя
Домашняя директория /home/alice
Оболочка /bin/bash
/etc/shadow
В файле хранятся:
хэши паролей
сроки действия пароля
политика паролей
Файл недоступен обычным пользователям, потому что содержит конфиденциальные данные

## 2. Права доступа chmod/chown

Итоговые права /srv/project/code: `chmod 770`, владелец alice:developers
- bob (в группе developers): не мог создать файл при 750 потому что потому что у группы не было права записи r-x
- carol (не в группе): не мог войти в директорию потому что нет прав ---

## 3. ACL

Задача: дать carol доступ на чтение без изменения группы.
Команда: `setfacl -m u:carol:r-x /srv/project/code`

Вывод getfacl после изменения:
carol может просматривать файлы
но не может записывать
# вставить вывод команды
setfacl -x u:carol /srv/project/code

## 4. sudo-политики

| Пользователь | Разрешено | Ограничение |
|---|---|---|
| alice | Всё (NOPASSWD) | Нет |
| bob | apt, apt-get | Только пакетный менеджер |
| carol | journalctl, cat /var/log/* | Только просмотр логов |
Реализуется принцип наименьших привилегий

## 5. PAM

Модуль pam_unix.so в /etc/pam.d/sudo: основной модуль аутентификации он проверяет пароль пользователя
Что означает required: если модуль не выполнен аутентификация завершается с ошибкой даже если другие прошли
Как добавить минимальную длину пароля 12 символов: в файле /etc/pam.d/common-password добавить minlen=12

## Выводы
Освоен принцип наименьших привилегий, который повышает безопасность системы
